# Sales-Data-Simulation-Dashboard-Analysis

🚀 Overview

This project focuses on generating realistic sales data to simulate a business environment and enable the development of data analysis and visualization solutions.

The main goal is to create a structured dataset that can be used for analytical purposes and dashboard creation, especially with tools like Power BI.

🧠 Project Objective

In real-world scenarios, access to clean and complete datasets is not always available.
This project demonstrates how to:

Generate synthetic but realistic data
Prepare datasets for analysis
Support business decision-making through data
🏗️ Data Generation

The dataset was programmatically generated using Python, simulating real sales operations.

🔹 Features of the dataset:
Thousands of sales records
Multiple products with different price ranges
Multiple cities (geographical distribution)
Variable quantities per transaction
Time-based data (daily sales over months)

The data was exported as a CSV file, making it easy to integrate with BI tools.

📊 Data Structure

The dataset includes the following fields:

date → Transaction date
product → Product name
city → Sales location
quantity → Number of items sold
unit_price → Price per item

This structure enables a wide range of analytical possibilities.

📈 Business Analysis Possibilities

With this dataset, it is possible to analyze:

Revenue trends over time
Best-selling products
Sales performance by city
Total revenue and sales volume
Average ticket per transaction

📊 Power BI Application

The generated dataset can be directly imported into Power BI to create interactive dashboards.

Example dashboards:
📈 Revenue evolution over time
🏆 Top-performing products
🌍 Sales by city
📊 KPI indicators (Total Revenue, Sales Volume, Average Ticket)
